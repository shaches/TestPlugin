# TestPlugin

This is a waypoint and coordinate management plugin for Minecraft (Paper 1.21+). The codebase was extensively vibe-coded (generated with the assistance of a Large Language Model). Note that this README file was also generated by an LLM.

## Features

  * **Personal Waypoints**: Save, update, and delete personal locations with associated names.
  * **Cross-Player Sharing**: Access other players' saved locations in a read-only capacity using the syntax `PlayerName:LocationName`.
  * **Command Injection**: Using a `#` prefix allows the insertion of saved coordinates into vanilla Minecraft commands.
  * **Storage**: Uses SQLite with HikariCP connection pooling for asynchronous database operations.
  * **Data Migration**: Automatically migrates legacy YAML location data to SQLite on startup.
  * **Customization**: Text and colors are configurable via `messages.yml` using the MiniMessage format.
  * **Visibility Cache**: A thread-safe visibility cache is used to prevent the leakage of vanished players' locations during asynchronous tab completion.
  * **Quota Limits**: Enforces configurable maximum location limits per player.

## Command Injection Syntax

The command interceptor replaces recognized location names with their corresponding X, Y, and Z coordinates within supported commands. Cross-world coordinate injection is blocked.

  * `#<location_name>` (Personal location)
  * `#<player_name>:<location_name>` (Cross-player location)

Example:
`/tp @p #home` translates to `/tp @p 100.50 64.00 200.25`

Supported Commands: `/tp`, `/teleport`, `/execute`, `/setblock`, `/fill`, `/clone`, `/particle`, `/summon`, `/spawnpoint`, `/setworldspawn`, `/goto`.

## Commands & Permissions

| Command | Description | Permission | Default |
| :--- | :--- | :--- | :--- |
| `/storeloc <name> <x> <y> <z>` | Saves a coordinate under a custom name. | `testplugin.basic` | True |
| `/setloc <name> <x> <y> <z>` | Overwrites an existing owned location. | `testplugin.basic` | True |
| `/deleteloc <name>` | Deletes a saved location. | `testplugin.basic` | True |
| `/listlocs [player]` | Lists saved coordinates. | `testplugin.basic` / `testplugin.others` | True / OP |
| `/goto <name>` <br> `/goto <player>:<name>` | Teleports to a saved location. | `testplugin.basic` / `testplugin.others` | True / OP |

Additional Permission: `testplugin.admin` allows bypassing the maximum location quota if the `admin-bypass-quota` setting is enabled.

## Configuration

`config.yml`:

```yaml
storage:
  max-locations-per-player: 50
database:
  max-pool-size: 10
permissions:
  admin-bypass-quota: true
```

## Technical Details

  * Built using Java 21 and the Paper 1.21 API.
  * Database operations are processed asynchronously via HikariCP and `CompletableFuture`.
  * Tab completion utilizes a main-thread updated `VisibilityCache` to comply with Bukkit API thread contracts during asynchronous execution.
  * The codebase includes unit tests written with JUnit 5 and Mockito.
